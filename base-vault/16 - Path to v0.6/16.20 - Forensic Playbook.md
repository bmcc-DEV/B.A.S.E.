# Playbook forense — 1 página (v0.6)

**Objetivo:** Evidence → contratos → Design (RP + STM32 Capstone/pins) + HIL CLI EXPERIMENTAL; engenheiro no loop.

## Setup (8–10 min)

```bash
git clone https://github.com/bmcc-DEV/B.A.S.E..git
cd B.A.S.E.
cargo build -p base-cli

# Gates
./examples/pilot/run.sh
./examples/pilot/run_t1_b2.sh

# STM32 (Capstone + pins + PCB draft)
./examples/pilot_stm32/run.sh

# HIL CLI (host-only)
base hil enumerate -o /tmp/hil_demo
base hil flash /dev/null --mock-flash -o /tmp/hil_demo   # ou ficheiro 1 byte
# opt-in: cargo build -p base-cli --features hil_usb,hil_programmer
```

## Mostrar ao cliente

1. **RP** — UART `0x40034000` + SPI opt-in.
2. **STM32** — Capstone `--disasm` @ página `0x40013000`; traces USART1; design ST → `STM32F103C8`; sch PA9/PA10 `NOT FABRICABLE`.
3. **Contratos** — replay + `proof_report.backend`.
4. **HIL CLI** — `base hil enumerate` → Simulated; `--mock-flash` → `mock_dry_run`; nunca production.
