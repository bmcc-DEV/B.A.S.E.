# Foundation — SpecterProbe Layers 1-4

## O que é

O SpecterProbe é o pipeline de **análise** que alimenta o B.A.S.E. Ele pega firmware bruto e produz modelos comportamentais estruturados (Device Genome).

Repositório: [bmcc-DEV/specterprobe](https://github.com/bmcc-DEV/specterprobe.git)

## Camadas

### Layer 1: Acquisition
- **Input:** Firmware .zip, boot.img, super.img
- **Processamento:** Extração de partições, parsing DTB, probe ADB
- **Output:** `AcquireOutput` — kernel, drivers, HALs, partições

### Layer 2: Lift
- **Input:** Kernel binary (ARM64)
- **Processamento:** Disassembly (Capstone) → CFG → LLVM IR
- **Output:** `LiftOutput` — funções, blocos, instruções, IR textual

### Layer 3: MMIO Discovery
- **Input:** Funções do lift + DTB
- **Processamento:** Scanner de acessos MMIO (ADRP/ADD/MOV propagation) → classificação por página
- **Output:** `MmioMap` — regiões, acessos, confidence

### Layer 4: Behavioral
- **Input:** MmioMap
- **Processamento:** Register map → init sequences → FSM → bitfields (XOR)
- **Output:** `BehaviorOutput` — device models

### 1. `lift::ir` — LLVM IR Real

**Problema:** O IR atual é simplificado (SSA inexistente, phi nodes ausentes, tipagem genérica).

**Implementação:**

```rust
enum SsaValue {
    Instruction(InstId),
    Constant(u64, Type),
    Phi(BasicBlockId, Vec<(SsaValueId, BasicBlockId)>),
}

struct FunctionIR {
    name: String,
    entry_block: BasicBlockId,
    blocks: HashMap<BasicBlockId, SsaBlock>,
    values: HashMap<SsaValueId, SsaValue>,
}

struct SsaBlock {
    id: BasicBlockId,
    instructions: Vec<SsaInstruction>,
    terminator: Terminator,
}
```

### 3. Cache Incremental

```rust
struct PipelineCache {
    cache_dir: PathBuf,
}

impl PipelineCache {
    fn get_or_compute<T: Serialize + Deserialize>(
        &self, key: CacheKey, compute: impl Fn() -> T
    ) -> T {
        let hash = sha256(&key);
        let path = self.cache_dir.join(hex::encode(hash));
        if path.exists() {
            deserialize_from_file(&path)
        } else {
            let result = compute();
            serialize_to_file(&path, &result);
            result
        }
    }
}
```

### 4. Melhorias nas Heurísticas

| Heurística Atual | Problema | Melhoria Proposta |
|-----------------|----------|------------------|
| Bitfields via XOR | Só detecta bits que alternam | Análise de padrões de escrita (mask tracking) |
| FSM fixa (Idle→Init→Running) | Não reflete hardware real | Inferência via N-gram + graph mining |
| Confidence heuristic | Arbitrária (DTB match = +0.4) | Baseada em taxa de acerto em cross-validation |
| Nome de registradores | Baseado em offset fixo | Context-aware naming (sequência de acessos) |
