# 📌 Master Plan — v0.3 Beyond the Wedge

> Meta de produto v0.3: *“Mesmo valor do v0.2, mais um eixo de confiança (FW real ou 2º alvo) + prova formal opcional na CI + drafts PCB/HIL um nível acima — ainda com engenheiro no loop.”*

## 1. Partida (pós-R6)

| Ativo | Estado |
|-------|--------|
| Piloto UART @ `0x40034000` | ✅ Case study + CI |
| Evidence / contracts / design / check / Ψ | ✅ REAL* no wedge |
| PCB / evolve | SCAFFOLD rotulado, opt-in |
| Z3 | Feature opcional, fora da CI default |
| HIL | Fora do workspace / experimental |
| `fw.bin` | Placeholder 60 B |

## 2. Camadas novas (L4–L6)

| Camada | Nome | Entrega |
|--------|------|---------|
| L4 | **Breadth** | FW mais real **ou** segundo alvo arquitetural (escolher 1 no S1) |
| L5 | **Formal** | Z3 nightly / job separado; prove report com backend flag |
| L6 | **Hardware loop** | Pin-aware mínimo + HIL template rodável sem board |

L1–L3 do v0.2 **não** regressam — `examples/pilot/run.sh` é gate permanente.

## 3. Decisão S1 (obrigatória no start)

Escolher **um** foco de amplitude (não ambos no mesmo sprint):

| Opção | Entrega | Risco |
|-------|---------|-------|
| **A — FW realista** | ELF/bin ARM mínimo com sequências MMIO UART reais; Capstone produz candidatos sem depender só de `--mmio-traces` | Médio (toolchain) |
| **B — 2º alvo** | Mesmo pipeline em arch/peripheral diferente (ex. STM32 USART @ faixa documentada) | Alto (escopo) |

Default recomendado: **A**, manter wedge único até S5.

## 5. Métricas de sucesso (v0.3)

1. [ ] Piloto v0.2 **ainda** verde na CI (zero regressão)
2. [ ] L4: Capstone ou 2º fixture gera ≥1 bloco sem override mentiroso
3. [ ] Job formal: ≥1 contrato Z3 SAT + 1 UNSAT na CI (não no `test` default se flaky)
4. [ ] Pin-aware: ≥1 componente com `pins` no path feliz; FPGA/MCU sem pinout continua documentado
5. [ ] HIL: build template sem HW; claim “flash real” só com probe detectado
6. [ ] Docs: Maturity Delta + COMMERCIAL sem novos overclaims
7. [ ] Demo: script único `examples/pilot/run_v03.sh` **ou** extensão documentada do `run.sh`

## 6. Anti-padrões

- [ ] Quebrar `run.sh` v0.2 “porque o novo piloto é melhor”
- [ ] Z3 obrigatório no `cargo test --workspace` sem cache/feature
- [ ] Chamar PCB pin-aware de fabricável
- [ ] HIL “passou” sem hardware e sem label EXPERIMENTAL
- [ ] Abrir 3 arquiteturas de uma vez
