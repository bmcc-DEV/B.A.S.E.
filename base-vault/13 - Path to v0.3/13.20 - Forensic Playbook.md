# Playbook forense — 1 página (v0.4)

**Objetivo da demo:** evidência → contratos → Reference Design no wedge UART (+ SPI opt-in), engenheiro no loop.

## Setup (5 min)

```bash
git clone https://github.com/bmcc-DEV/B.A.S.E..git
cd B.A.S.E.
cargo build -p base-cli
./examples/pilot/run.sh          # gate UART (v0.2/v0.3)
./examples/pilot/run_t1_b2.sh    # v0.4 — UART + SPI0 @ 0x4003c000
# opcional: cargo test -p base-core --features solver_z3 --lib smt   # requer libz3
```

Artefatos: `examples/pilot/out/` · `out_t1_b2/` · `CASE_SUMMARY*.md` · hashes `SHA256SUMS*`.

## O que mostrar ao cliente (nessa ordem)

1. **Evidence** — UART `0x40034000` (+ SPI `0x4003c000` no path B2).
2. **Contratos** — replay feliz + fixture que **falha**; `proof_report.json` com `backend`.
3. **Design** — MCU RP2040 + BOM; contratos ≥70% no wedge.
4. **Limites** — PCB = `NOT FABRICABLE`; FW host ≠ silício; HIL = EXPERIMENTAL (sem flash sozinho).

Case study: [[12 - Path to Real/12.20 - Pilot Case Study]] · plano [[14 - Path to v0.4/14.00 - Index]].

## Não dizer

- “Substitui ASIC” / “PCB fabricável” / “prova Z3 always-on” / “HIL flasheou sozinho”.
