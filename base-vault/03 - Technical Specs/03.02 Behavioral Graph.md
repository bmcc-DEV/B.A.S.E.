# Behavioral Graph

## Estrutura

```rust
struct BehavioralGraph {
    nodes: Vec<GraphNode>,
    edges: Vec<GraphEdge>,
    metadata: GraphMetadata,
}

enum GraphNode {
    SoC { name: String, compatible: Vec<String> },
    Block { id: BlockId, kind: BlockKind, base: u64 },
    Register { offset: u32, name: String, purpose: RegisterPurpose },
    State { block: BlockId, name: String },
    Value { address: u64, value: u32, context: String },
    Function { address: u64, name: String },
    Driver { name: String, interface: String },
    Interrupt { vector: u8, handler: u64 },
    DmaChannel { id: u8, src: u64, dst: u64 },
    Timing { operation: String, min: u64, max: u64, avg: u64 },
}

enum GraphEdge {
    Contains,
    Controls,
    TransitionsTo,
    Accesses,
    Triggers,
    Calls,
    MapsTo,
    HasTiming,
}

struct GraphMetadata {
    node_count: usize,
    edge_count: usize,
    confidence: f64,
    source: String,
}
```

## Exemplo

```yaml
nodes:
  - type: SoC
    id: soc_0
    name: "Power Mac G5"

  - type: Block
    id: gpu_0
    kind: Gpu
    base: 0x10000000

  - type: Register
    id: reg_gpu_ctrl
    offset: 0x00
    name: control
    purpose: Control

  - type: State
    id: state_idle
    block: gpu_0
    name: Idle

  - type: State
    id: state_active
    block: gpu_0
    name: Active

edges:
  - from: soc_0
    to: gpu_0
    kind: Contains
    weight: 1.0

  - from: gpu_0
    to: reg_gpu_ctrl
    kind: Controls
    weight: 1.0

  - from: state_idle
    to: state_active
    kind: TransitionsTo
    weight: 0.95
    metadata:
      condition: "control.write(0x01)"
      latency_avg: 1.2us
```

## Serialização

O grafo é serializado em JSON e pode ser importado no Neo4j:

```cypher
MATCH (b:Block {kind: 'Gpu'})-[:Controls]->(r:Register)
WHERE r.purpose = 'Control'
MATCH (b)-[:Contains]->(s:State)
RETURN b.name, r.offset, s.name
```
