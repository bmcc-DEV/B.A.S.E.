## Problema

Hoje o Contract Solver faz **matching** (`requires ⊆ provides`). Isso é checking, não proving.

```yaml
# Checking: "este componente tem DMA?"
requires: dma  →  RP2350: ✅ (8 channels)

# Mas não prova: "o sistema nunca vai deadlock?"
```

## Solução

**SMT proving** via Z3:

1. **Latência máxima** ≤ latência requerida
2. **Ordenação de eventos** sempre respeitada
3. **Sem deadlock** entre IRQs e DMAs
4. **Barramento nunca satura**

## Tradutor Contract → SMT-LIB2

```rust
fn contract_to_smt(contract: &SequenceContract) -> String {
    let mut smt = String::new();

    // Declara variáveis de tempo simbólicas
    for (i, step) in contract.steps.iter().enumerate() {
        smt.push_str(&format!("(declare-const t_{} Int)\n", i));
    }

    // Ordem causal: t_i < t_{i+1}
    for i in 0..contract.steps.len() - 1 {
        smt.push_str(&format!("(assert (< t_{} t_{}))\n", i, i + 1));
    }

    // Latência total: t_last - t_first ≤ max
    let last = contract.steps.len() - 1;
    smt.push_str(&format!(
        "(assert (<= (- t_{} t_0) {}))\n",
        last, contract.max_total_ns
    ));

    // Latência entre passos: t_{i+1} - t_i ≤ max_step
    if contract.max_step_ns > 0 {
        for i in 0..contract.steps.len() - 1 {
            smt.push_str(&format!(
                "(assert (<= (- t_{} t_{}) {}))\n",
                i + 1, i, contract.max_step_ns
            ));
        }
    }

    smt.push_str("(check-sat)\n");
    smt
}
```

## Exemplo de Prova

```lisp
; Provar: DMA sequence com latência total ≤ 5µs (5000ns)
(declare-const t0 Int)  ; mmio_write
(declare-const t1 Int)  ; dma_start
(declare-const t2 Int)  ; dma_complete
(declare-const t3 Int)  ; irq

(assert (< t0 t1))      ; write antes de dma_start
(assert (< t1 t2))      ; dma_start antes de dma_complete
(assert (< t2 t3))      ; dma_complete antes de irq
(assert (<= (- t3 t0) 5000))  ; latência total ≤ 5µs
(assert (<= (- t1 t0) 1000))  ; step 1 ≤ 1µs
(assert (<= (- t2 t1) 3000))  ; step 2 ≤ 3µs
(assert (<= (- t3 t2) 1000))  ; step 3 ≤ 1µs

(check-sat)
; => sat ✓ (prova: a sequência cabe em 5µs)

(get-model)
; => t0=0, t1=100, t2=2300, t3=2500  (exemplo)
```

## Invariantes

```lisp
; Provar: nunca ocorre IRQ sem DMA_COMPLETE antes
(declare-const t_irq Int)
(declare-const t_dma_complete Int)

(assert (and
    (< t_dma_complete t_irq)          ; IRQ depois do DMA
    (<= (- t_irq t_dma_complete) 500) ; IRQ até 500ns depois
))
(check-sat)
; => sat ✓ (invariante mantido)
```

```lisp
; Provar: ausência de deadlock
(declare-const t_dma1 Int)
(declare-const t_dma2 Int)
(declare-const t_irq1 Int)
(declare-const t_irq2 Int)

(assert (and
    (< t_dma1 t_dma2)      ; DMAs em sequência
    (< t_irq1 t_irq2)      ; IRQs em sequência
    (< t_dma2 t_irq1)      ; segundo DMA antes do primeiro IRQ
    (< (- t_irq2 t_dma1) 10000)  ; tudo em 10µs
))
(check-sat)
; => sat ✓ (sem deadlock)
```
