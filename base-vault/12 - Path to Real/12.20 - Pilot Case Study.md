# 🧪 Pilot Case Study — UART MMIO Wedge (v0.2)

> Case study **auditável** do Path to Real. Wedge **sintético** (não um SoC de cliente).  
> Fixtures: [`examples/pilot/`](../../../examples/pilot/) · Smoke: `./examples/pilot/run.sh`

## 1. Contexto do hardware

| Campo | Valor |
|-------|-------|
| Classe | Peripheral MMIO estilo UART PL011 / MCU ARM-like |
| Base | `0x40034000` |
| Evidência | `mmio.json` (4 acessos) + `trace.csv` Saleae-like |
| FW | `fw.bin` **placeholder** (60 B) — a semântica vem dos traces |
| Arquitetura alvo do design | MCU barato com `uart` (tipicamente **RP2040**, ~$0.70) |
| Fora de escopo | GPU, SerDes HS, Power Mac G5, Xbox, Alpha |

**Por quê este wedge:** MMIO estável, tipos WRITE/READ/IRQ, contratos SAT/UNSAT, cabem Capstone/heurística e Saleae CSV.

## 2. Como reproduzir (copy-paste)

```bash
git clone https://github.com/bmcc-DEV/B.A.S.E..git
cd B.A.S.E.
cargo build -p base-cli
./examples/pilot/run.sh
# → examples/pilot/out/ + CASE_SUMMARY.md
```

Verificação de integridade:

```bash
cd examples/pilot && sha256sum -c SHA256SUMS
```

Pipeline orquestrado (PCB/evolve opt-in):

```bash
./target/debug/base pipeline examples/pilot/fw.bin \
  --trace examples/pilot/trace.csv \
  --new-trace examples/pilot/trace_slow.csv \
  -o /tmp/base_pipeline_pilot
# SUMMARY.md sem PCB; com --pcb gera 06_pcb/ NOT FABRICABLE
```

## 3. Resultados típicos (R0–R5)

| Artefato | Resultado esperado |
|----------|-------------------|
| Evidence DB | ≥4 entries MMIO alinhadas a `0x40034000` |
| HardwareSpec | 1 bloco UART após `--classify uart` |
| Tension Ψ | `tension_report.json` — tipicamente Ψ≈0.32, confidence≈76%, `inconclusive` |
| Reference Design | CPU MCU real (≠ TBD), interface `uart`, **3/3** contratos, BOM > 0 |
| Synth | Assignment MCU sob `--max-bom-cost 80` |
| Prove SAT | `proved: true` em contratos manuais + BIR |
| Prove UNSAT | `impossible_total` → `proved: false` |
| Replay pass | 0 violations em `trace.csv` |
| Replay fail | Violations em `trace_fail.csv` |
| Check skip | `NO_NEW_TRACE` / `comparison_mode=skipped` (sem self-pass) |
| Check dual | `TIMING_VIOLATION` vs `trace_slow.csv` |
| FW host | `make host` + `firmware_host` exit 0 (**≠ silício**) |

Goldens: `examples/pilot/expected/` (fields allowlist, event_graph, notes de design).

## 4. O que falhou / limitações honestas

- `fw.bin` **não** é firmware de produto — só ancora o path CLI.
- Classificação UART exige `--classify` + traces (sem ML mágico).
- Capstone puro no placeholder de 60 B quase não gera MMIO — daí `--mmio-traces`.
- PCB/evolve **fora** do smoke de propósito; draft KiCad = `NOT FABRICABLE`.
- Ψ no piloto fica `inconclusive` (pouca massa estrutural) — correto, não “hackar” confidence.
- Z3 não é default; prove simbólico basta para SAT/UNSAT do wedge.

## 5. Valor para um engenheiro de fora

Em ~2 minutos de smoke você obtém:

1. Spec + evidência serializáveis  
2. Contratos que passam **e** falham de propósito  
3. Design com MCU + BOM estimada  
4. Relatório de validação que **recusa** auto-confirmação  

Isso é **assistência** de RE comportamental — não substituição drop-in de ASIC.

## 6. Addendum v0.3 / S1 — Capstone

`fw.bin` agora é blob **AArch64-LE** gerado por `gen_fw.py` (`adrp`+`str`/`ldr` @ `0x40034000`).  
`base analyze fw.bin --disasm` resolve a página UART **sem** `--mmio-traces` (scanner Adrp). O smoke ainda usa traces+`--classify uart` para o path feliz de contratos.
