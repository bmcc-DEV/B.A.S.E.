# 📌 Master Plan — Tornar B.A.S.E. real e funcional

> Meta de produto v0.2: *“Envie firmware + (opcional) trace → receba Evidence DB, contratos, Reference Design e relatório de prova/replay que um engenheiro consegue auditar.”*

## 1. Norte

### É (v0.2)

- Motor de **engenharia reversa comportamental assistida**
- Saída principal: [[10 - B.A.S.E. v3.1 Evidence-Driven/10.05 - Reference Design|Reference Design]] + contratos + Ψ
- PCB/FW: **rascunhos hospedáveis / compiláveis em host**, marcados como draft

### Não é (ainda)

- Substituição drop-in de ASIC industrial
- Layout KiCad pronto para fábrica
- Firmware produção em alvo silício sem revisão humana

Ver anti-overclaim em [[12.02 - Maturity Matrix]].

---

## 2. Estratégia em 3 camadas

| Camada | Nome | Entrega |
|--------|------|---------|
| L1 | **Truth** | Evidência + contratos + replay/prove sem mentira |
| L2 | **Design** | Mapping + BOM + Reference Design consistentes |
| L3 | **Drafts** | FW host-CI + PCB S-expr honesto / DRC opcional |

Sprints [[12.10 - Sprint R0 Baseline|R0]]–[[12.16 - Sprint R6 Pilot|R6]] cobrem L1→L3.

---

## 3. Wedge piloto (obrigatório)

Escolher **1** binário e documentar tudo nele. Candidatos:

| Prioridade | Alvo | Por quê |
|------------|------|---------|
| 1 | Bootloader / FW MCU ARM Cortex-M com MMIO conhecido (ex. RP2040/STM32 demo) | Capstone + faixas MMIO estáveis |
| 2 | SoC ARM64 pequeno (image parcial) | Já usado no disasm path |
| ❌ adiar | Power Mac G5 / Xbox / Alpha | Escopo de pesquisa, não produto |

Cada artefato do piloto vai em `examples/pilot/` + nota no vault (R6).

## 5. Métricas de sucesso (v0.2)

Marcar ✅ só se todos passarem:

1. ✅ **Reprodutibilidade:** `examples/pilot/run.sh` + `SHA256SUMS` + goldens `expected/`
2. ✅ **Contratos:** ≥1 sequência temporal verificável via `replay` no piloto
3. ✅ **Prove:** SAT vs UNSAT sem Z3 (fixture `contracts.unsat.yaml`)
4. ✅ **Design:** 0 `"TBD"` / `"unassigned"` no path feliz (MCU + ≥70% contratos)
5. ✅ **FW:** CI `Pilot FW host gate` (`firmware_host` exit 0)
6. ✅ **Docs:** README + [[12.02 - Maturity Matrix]] + [[12.20 - Pilot Case Study]]; sem claim drop-in
7. ✅ **Ψ:** `tension_report.json` em analyze

## 6. Anti-padrões (bloquear)

- [ ] Novos comandos CLI sem teste + fixture
- [ ] Inflar confidence sem mudança estrutural
- [ ] Gerar KiCad e chamar de “pronto para fab”
- [ ] Cases G5/Xbox/Alpha no README como “suportados”
- [ ] Dependência default em ML/ONNX para o núcleo
