# 📌 Master Plan — Tornar B.A.S.E. real e funcional

> Meta de produto v0.2: *“Envie firmware + (opcional) trace → receba Evidence DB, contratos, Reference Design e relatório de prova/replay que um engenheiro consegue auditar.”*

## 1. Norte

### É (v0.2)

- Motor de **engenharia reversa comportamental assistida**
- Saída principal: [[10 - B.A.S.E. v3.1 Evidence-Driven/10.05 - Reference Design|Reference Design]] + contratos + Ψ
- PCB/FW: **rascunhos hospedáveis / compiláveis em host**, marcados como draft

### Não é (ainda)

- Substituição drop-in de ASIC industrial
- Layout KiCad pronto para fábrica
- Firmware produção em alvo silício sem revisão humana

Ver anti-overclaim em [[12.02 - Maturity Matrix]].

---

## 2. Estratégia em 3 camadas

| Camada | Nome | Entrega |
|--------|------|---------|
| L1 | **Truth** | Evidência + contratos + replay/prove sem mentira |
| L2 | **Design** | Mapping + BOM + Reference Design consistentes |
| L3 | **Drafts** | FW host-CI + PCB S-expr honesto / DRC opcional |

Sprints [[12.10 - Sprint R0 Baseline|R0]]–[[12.16 - Sprint R6 Pilot|R6]] cobrem L1→L3.

## 5. Métricas de sucesso (v0.2)

Marcar ✅ só se todos passarem:

1. **Reprodutibilidade:** `base pipeline <fixture> --disasm` idêntico em CI (hash YAML estável ou golden allowlist).
2. **Contratos:** ≥1 sequência temporal verificável via `replay` no piloto.
3. **Prove:** distinguie SAT vs UNSAT sem feature Z3; com Z3 opcional na CI nightly.
4. **Design:** 0 campos `"TBD"` / `"unassigned"` no path feliz do piloto.
5. **FW:** job CI `make -C output/04_fw host && ./firmware_host`.
6. **Docs:** README e vault com matriz [[12.02 - Maturity Matrix]]; nenhum claim drop-in.
7. **Ψ:** tension report emitido em analyze/design.
