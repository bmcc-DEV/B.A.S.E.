# ✅ Acceptance Criteria — artefatos

Cada artefato tem **formato**, **invariantes** e **teste**.

## Evidence DB (`evidence_db.yaml`)

- [ ] Só fatos (MMIO/IRQ/DMA) — sem `kind: Gpu` dentro
- [ ] Cada entry tem `id`, `evidence_type`, `context`
- [ ] `source` aponta para o binário/fixture
- [ ] Teste: round-trip YAML; count ≥ N da fixture

## HardwareSpec (`hardware_spec.yaml`)

- [ ] `blocks[].base_address` alinhado a evidence
- [ ] `confidence` = f(evidence) [[12.11 - Sprint R1 Analyze|R1]], não monótona fake
- [ ] `--classify` altera `kind` de forma determinística
- [ ] Golden test com allowlist de campos voláteis

## BIR + Contracts

- [ ] BSL → BIR via `base bir --compile`
- [ ] `bir_to_sequence_contracts` ≥1 contrato no piloto
- [ ] `replay` com `--bir` ≠ lista vazia
- [ ] `prove` reporta UNSAT quando `max_total < n-1` (strict)

## Reference Design

- [x] `architecture.cpu.part ≠ TBD|unassigned` no path feliz (piloto: RP2040)
- [x] `contracts.total > 0` e reasons em violations (part no detalhe)
- [x] `bom.estimated_cost` soma `price_1k` conhecidos
- [x] Status: `contracts_satisfied | contracts_mostly_satisfied | needs_review | unmapped` (≥70% gate)

## Validation report

- [ ] Sem `--new-trace`: pipeline **pula** check ou falha em modo `--strict`
- [ ] Com traces: pass rate + lista de ops
- [ ] Ψ (tension) no summary JSON

## Firmware draft

- [ ] `main.c` + `Makefile` com target `host`
- [ ] Compila sem toolchain ARM
- [ ] Sem símbolos Pico SDK / NVIC sem stub

## PCB draft

- [ ] Header comentário: `engineering_draft — NOT FABRICABLE`
- [ ] BOM manufacturer pode ser TBD **só** se note existir
- [ ] Fora do `pipeline` default OU atrás de `--pcb`
