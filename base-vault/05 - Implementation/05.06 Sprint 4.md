# Sprint 4 — Synthetic Firmware ✅

## Objetivo

Gerar bootloader, HAL e drivers para o novo hardware.

## Tarefas

- [x] Gerador de bootloader (inicialização de clock, DRAM, MMU)
- [x] Gerador de HAL de tradução MMIO
- [x] Compensação de timing (delays artificiais)
- [x] Tabela de interrupções
- [x] Gerador de drivers bare-metal
- [x] Suporte a Zephyr RTOS

## Resultado

| Módulo | Arquivos | Status |
|--------|----------|--------|
| bootloader | `base-fw/src/bootloader.rs` | ✅ Gera C: system_clock_init, dram_init, mmu_init, load_firmware, jump_to_entry — corrigido para Cortex-M33 (sem asm AArch64) |
| hal | `base-fw/src/hal.rs` | ✅ `mmio_read()` + `mmio_write()` com tabela de tradução + handlers por bloco (GPU, Audio, DMA) |
| timing | `base-fw/src/timing.rs` | ✅ `timing_compensate()` com delays por bloco + `original_timing[]` table |
| irq | `base-fw/src/irq.rs` | ✅ `irq_translation[]` table + `irq_handler()` + `irq_init()` — corrigido `__get_IPSR()` |
| drivers | `base-fw/src/drivers.rs` | ✅ 5 drivers (SPI, I2C, UART, USB, GPIO) + Makefile + linker script |
| zephyr | `base-fw/src/zephyr.rs` | ✅ Kconfig, devicetree overlay, CMakeLists.txt, prj.conf |

## Critérios de Aceite

1. ✅ Bootloader inicializa hardware alvo e faz transição — código C gerado com init sequence
2. ⚠️ HAL < 200ns overhead — implementado, não benchmarkado (linear search na tabela)
3. ✅ Timing compensation — delays injetados baseados no perfil original
4. ✅ Interrupções mapeadas — tabela de tradução com init
