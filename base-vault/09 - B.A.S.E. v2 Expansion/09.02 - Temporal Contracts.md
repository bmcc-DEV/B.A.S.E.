## Contratos Temporais

```rust
struct BirContract {
    must_occur_before: Vec<CausalOrder>,  // A -> B (A deve ocorrer antes de B)
    latency: Vec<BirLatencyConstraint>,    // latência min/max por evento
    window_ns: Option<u64>,               // janela total máxima
    jitter_ns: Option<u64>,              // variação máxima
    repetition_rate: Option<u64>,         // frequência mínima
}
```

## Novas Capacidades para Port de OS

### HAL com MMU Page Table (O(1))
Arquivo: `base-fw/src/hal_mmu.rs`

Gera código C que usa **MMU page table** em vez de busca linear O(n):

```c
// Antes (O(n)):
for (int i = 0; mmio_translation[i].base != 0; i++)
    if (addr >= mmio_translation[i].base && ...)

// Depois (O(1) via MMU):
mmu_l2_table[addr >> 12] = output_page | 3;  // hardware traduz
MMIO_WRITE(addr, value);  // sem lookup em runtime
```

### Device Tree Linux (.dts)
Arquivo: `base-fw/src/devicetree.rs`

Gera Device Tree Linux a partir do HardwareSpec:

```dts
/dts-v1/;
/ {
    model = "B.A.S.E. — Behavioral ASIC Reconstruction";
    soc {
        uart0@a9bf0000 {
            compatible = "ns16550a";
            reg = <0x0 0xa9bf0000 0x0 0x1000>;
            interrupts = <0x20 0x4>;
        };
    };
};
```

### Contract Verifier Integrado

O `ContractVerifier` pode ser usado pelo `base-check` para validar que o hardware gerado respeita os contratos temporais do original — essencial para garantir que o OS port não quebre por timing.
