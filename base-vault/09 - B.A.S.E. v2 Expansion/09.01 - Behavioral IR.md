# Fase 1 — Behavioral IR (BIR)

**Esforço:** 6 semanas | **Risco:** Médio | **Inovação:** ★★★★★

> O LLVM IR do hardware. Uma linguagem intermediária própria, serializável, verificável, versionável e independente de arquitetura.

## Conceito

O BIR é a representação canônica de hardware no B.A.S.E. v2. Todo o pipeline — do SpecterProbe ao Constraint Solver — opera sobre BIR.

## Tarefas

- [ ] Definir gramática BSL (Behavior Specification Language)
- [ ] Implementar parser BSL → BIR AST
- [ ] Implementar validador BIR (tipos, ciclos, referências)
- [ ] Implementar serializador BIR → YAML
- [ ] Implementar deserializador YAML → BIR
- [ ] Versionamento semântico de schemas BIR (v1→v2 migration path)
- [ ] Testes de snapshot com 10+ device descriptions em BSL

## AST BIR (Rust)

```rust
struct BirDevice {
    name: String,
    base_address: u64,
    registers: Vec<BirRegister>,
    events: Vec<BirEvent>,
    interrupts: Vec<BirInterrupt>,
    timing: BirTiming,
    contracts: Vec<BirContract>,
}

struct BirRegister {
    name: String,
    offset: u32,
    access: BirAccess,
    width: u8,
    reset_value: Option<u64>,
    bitfields: Vec<BirBitfield>,
}

enum BirAccess { Read, Write, ReadWrite, WriteOnly }

struct BirEvent {
    name: String,
    trigger: BirTrigger,
    timing: Option<BirLatencyRange>,
}

struct BirContract {
    must_occur_before: Vec<(String, String)>, // (event, target)
    latency: Vec<BirLatencyConstraint>,
    window: Option<u64>, // max timeout ns
}
```

## BSL Exemplo

```bsl
device GPU @ 0x10000000 {
    registers {
        CONTROL @ 0x00: rw;
        STATUS   @ 0x04: ro;
        BUF_ADDR @ 0x08: wo;
        LENGTH   @ 0x0C: rw;
    }

    events {
        DMA_START:   write CONTROL[0] = 1;
        DMA_COMPLETE: read STATUS[7]  = 1;
    }

    interrupts {
        IRQ_GPU: level high;
    }

    timing {
        dma_start:  100ns..400ns;
        dma_xfer:   50ns..200ns per_word;
        complete:   <= 500ns;
    }
}
```

## Relação com BSL

- **BIR** = formato interno (AST Rust, serializável YAML)
- **BSL** = linguagem textual que o engenheiro escreve
- O parser BSL compila para BIR
- BIR pode ser serializado como YAML para debug
