## Interpretador BIR

```rust
struct BirInterpreter {
    device: BirDevice,
    state: HashMap<String, u64>,
    time: u64,  // ns simulados
    trace: Vec<TraceEvent>,
}

impl BirInterpreter {
    fn write_register(&mut self, name: &str, value: u64) {
        self.state.insert(name.to_string(), value);
        self.trace.push(TraceEvent {
            timestamp_ns: self.time,
            event_type: MmioWrite,
            address: self.device.register_base(name),
            value: Some(value),
        });

        // Verifica se a escrita dispara eventos
        for event in &self.device.events {
            if event.trigger.matches_write(name, value) {
                self.fire_event(&event.name);
            }
        }
    }

    fn fire_event(&mut self, event: &str) {
        // Avança o tempo simulado pela latência do evento
        if let Some(latency) = self.device.event_latency(event) {
            self.time += latency;
        }

        // Verifica contratos
        for contract in &self.device.contracts {
            contract.verify(&self.trace);
        }
    }
}
```

## Pipeline do Twin

```text
Trace Original
     │
     ▼
┌────────────────┐
│  BirInterpreter │──► Simula MMIO writes/reads
│  + Temporal     │──► Avança tempo por latências
│  Contracts      │──► Dispara eventos
└────────┬───────┘
         │
         ▼
┌────────────────┐
│  base-check    │──► Compara: Twin gerou mesmos eventos?
│  validate()    │──► Timing respeitou contratos?
└────────────────┘
         │
         ▼
   "94% pass — pode gerar PCB"
   ou
   "Falhou: IRQ_GPU com latência 600ns (max 500ns)"
```

base twin run hardware_spec.yaml            # Executa twin
base twin run hardware_spec.yaml --trace original.pcap  # Valida contra trace
base twin interactive                        # Modo interativo (REPL)
