# Reconstruct — `--continuous` e estagnação (L18)

> **Não** é auto-fix completa. O loop só refine estrutural local a partir do `HardwareSpec` já analisado.

## Comportamento

| Parada | Quando |
|--------|--------|
| `converged` | `pass_rate >= --threshold` |
| `stagnated` | 0 mudanças estruturais na iteração (sem novas traces/evidência) |
| `max_iterations` | teto atingido |

`--continuous` **só** eleva o teto para `CONTINUOUS_ITERATION_CAP` (1000). Ainda para em converge/estagnação.

## CLI

```bash
base reconstruct path/to/hardware_spec.yaml -o out/recon
base reconstruct spec.yaml --continuous --threshold 0.99 -o out/recon
# convergence_report.json inclui: stop_reason, stagnated, auto_fix_complete=false
```

## O que o refine faz / não faz

| Faz | Não faz |
|-----|---------|
| Nomear regs sem nome | Inventar MMIO novo |
| Reclassificar `Unknown` | Flash / HIL / silício |
| Preencher timing placeholder zero | Garantir threshold sem evidência |

## Relatório

`convergence_report.json`:

- `stop_reason`: `converged` \| `stagnated` \| `max_iterations`
- `stagnated`: bool
- `auto_fix_complete`: sempre `false` (honestidade de produto)
