# SOW checklist — Port / diagnóstico industrial (v0.5)

> Copiar para o contrato. **Não** é SaaS turnkey. Humano no loop obrigatório.
> **v0.6 checklist:** [[16 - Path to v0.6/16.21 - SOW Industrial Checklist]].

## 1. Partes e foco

- **Cliente:** …
- **Prestador:** …
- **Alvo:** firmware / SoC / interface (ex.: USART STM32 @ `0x40013800`, UART RP @ `0x40034000`)
- **Ferramenta:** B.A.S.E. AGPLv3 ou licença comercial

## 2. Escopo (inclui) — checklist

- [ ] A — `analyze` + Evidence DB do blob acordado
- [ ] B — Contratos / replay quando traces existirem (`proof_report.backend` honesto)
- [ ] C — Reference Design (MCU/BOM) — draft; preferência de fabricante se acordada
- [ ] D — Relatório de viabilidade (Ψ / gaps / confiança)
- [ ] E — Demo reprodutível: `run.sh` e, se no escopo, `pilot_stm32/run.sh`
- [ ] F — Workshop de handoff (opcional, X horas)

## 3. Não-escopo (exclui por default)

- [ ] PCB/Gerber **fabricável** ou DRC de fábrica
- [ ] Firmware de produção em silício / certificação
- [ ] Flash HIL automático sem probe + `BASE_HIL_ALLOW_FLASH` + aprovação escrita
- [ ] Claim “HIL production” / `mode=production`
- [ ] `libusb` / probe USB como entrega de CI do Cliente
- [ ] Prova Z3 universal (só sob `solver_z3` / `formal.yml`)
- [ ] Drop-in ASIC / pin-compat elétrico completo garantido
- [ ] Turnkey SaaS / 24×7
- [ ] Substituir o gate RP pelo wedge STM32 como única baseline

## 4. Humano no loop

Decisões de arquitetura, BOM final, layout e release exigem **aprovação de engenheiro nomeado**. A ferramenta acelera evidência; não assina produto.

## 5. Entregáveis

- Pacote `out/` (YAML/JSON/MD) + hashes  
- Relatório de viabilidade  
- Links: [[15.20 - Forensic Playbook]] · [[12 - Path to Real/12.20 - Pilot Case Study]]

## 6. Aceite

1. Smoke `./examples/pilot/run.sh` (+ `run_t1_b2.sh` se SPI no escopo) verde no Prestador.  
2. Bloco alvo → Evidence + path de design **ou** bloqueio honesto.  
3. Se STM32 no escopo: `./examples/pilot_stm32/run.sh` ou relatório de bloqueio.  
4. Relatório **sem** claims da seção 3.  
5. Até N ciclos de feedback; além = change order.

## 7. HIL (se contratado à parte)

| Item | Default |
|------|---------|
| Enumerate USB | Feature `hil_usb` no ambiente do Prestador — **não** obrigação de CI do Cliente |
| Flash | `mock_dry_run` ou CMD externo EXPERIMENTAL; nunca “production” |
| Probe | Cliente fornece hardware + ferramenta (`BASE_HIL_PROGRAMMER_CMD`) |

## 8. Preço / prazo (preencher)

- Fase 1 (diagnóstico): ___ / ___ semanas  
- Fase 2 (port assistido): change order  
- Add-on HIL lab: ___  

## 9. Licença / IP

Código B.A.S.E.: LICENSE/acordo. Artefatos do estudo: Cliente (salvo upstream).
