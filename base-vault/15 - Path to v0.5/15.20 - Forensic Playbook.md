# Playbook forense — 1 página (v0.5)

**Objetivo da demo:** Evidence → contratos → Reference Design em **dois wedges** (RP + STM32), engenheiro no loop; HIL só como EXPERIMENTAL.

## Setup (5–8 min)

```bash
git clone https://github.com/bmcc-DEV/B.A.S.E..git
cd B.A.S.E.
cargo build -p base-cli

# Gates (nunca vermelhos)
./examples/pilot/run.sh              # UART RP @ 0x40034000
./examples/pilot/run_t1_b2.sh        # UART + SPI0 @ 0x4003c000

# Segundo SoC (opt-in v0.5)
./examples/pilot_stm32/run.sh        # USART1 @ 0x40013800 → STM32F103C8

# HIL host-only (CI default — sem USB)
cargo test -p base-hil
# opt-in local: --features hil_usb | hil_programmer
```

Artefatos: `examples/pilot/out/` · `out_t1_b2/` · `pilot_stm32/out/` · `CASE_SUMMARY*.md` · `SHA256SUMS*`.

## O que mostrar ao cliente (nessa ordem)

1. **Evidence RP** — UART `0x40034000` (+ SPI `0x4003c000` no B2).
2. **Evidence STM32** — USART1 regs @ `0x40013800` (bloco analyze @ página `0x40013000`); design com `--preferred-manufacturer STMicroelectronics`.
3. **Contratos** — replay feliz + falha; `proof_report.json` com `backend`.
4. **Design** — RP2040 / STM32F103C8 + BOM; contratos ≥70% no wedge.
5. **Limites HIL** — enumerate/`hil_usb` opt-in; flash = `mock_dry_run` ou `experimental_external_cmd` (nunca production); CI sem libusb.
