# Component Database Schema

## Formato YAML

Cada componente é um arquivo YAML no diretório `04 - Component Database/catalog/`.

```yaml
# Schema de entrada do Component DB
part: "RP2350A"
manufacturer: "Raspberry Pi"
description: "Dual Cortex-M33 + dual RISC-V, PIO, USB"
category: mcu
package: "QFN-56"

features:
  cpu:
    cores: 4
    max_mhz: 150
  memory:
    sram: "520K"
    flash: "4M (external QSPI)"
  peripherals:
    dma: 8
    pio: 2
    i2c: 2
    spi: 2
    uart: 2
    usb: 1
    timer: 4
    pwm: 8
    adc: 4

power:
  vcore: "1.1V @ 150mA"
  vccio: "3.3V @ 50mA"
  vusb: "5V"

pins:
  - number: 1
    name: GP0
    functions: [gpio, spi0_rx, uart0_tx, pio0_d0]
  - number: 2
    name: GP1
    functions: [gpio, spi0_tx, uart0_rx, pio0_d1]

emulation:
  max_pio_instructions: 32
  pio_sm_count: 8
  dma_chainable: true
```

## Campos de Busca

| Campo | Tipo | Descrição |
|-------|------|-----------|
| `part` | string | Número de parte |
| `manufacturer` | string | Fabricante |
| `category` | enum | mcu, cpu, dsp, fpga, pmic, memory, connectivity |
| `features.cpu.cross` | int | Núcleos mínimos |
| `features.peripherals.*` | int | Quantidade mínima de cada periférico |
| `power.*` | string | Tensões suportadas |
| `package` | string | Tipo de package |
| `emulation.*` | bool/int | Capacidades de emulação |
