# Port Package — mapa + fósseis + atlas

```bash
base port package hardware_spec.yaml \
  --evidence evidence_db.yaml \
  --tension tension_report.json \
  --target-hal hal_tauros_aarch64 \
  --hal-stub \
  --dtb vendor_boot.img \
  --flash-cfg flash.cfg \
  -o out/port_package/
```

```bash
# Inventário OS-port (CPU/GIC/timer/UART/…) a partir do Device Tree
base port platform vendor_boot.img --flash-cfg flash.cfg -o out/platform/
```

## Saídas

| Ficheiro | Conteúdo |
|----------|----------|
| `address_driver_map.yaml` | source MMIO → `hal_id` + strategy (MMU/TRAP/PIO/STUB) |
| `fossil_inventory.yaml` | regs/blocos não observados, Unknown, alto Ψ, orphans |
| `PORT_PACKAGE.md` | atlas humano (checklist wrap vs rewrite) |
| `port_package.yaml` | pacote completo |
| `hal_mmio_stub.c` | stub C host (`--hal-stub`) |
| `PLATFORM_INVENTORY.md` | CPU/GIC/timer/MMU/DRAM/UART/GPIO/PMIC/UFS/GPU/DTB |
| `platform_inventory.yaml` | mesmo inventário em YAML |

## Classes obrigatórias (OS port)

`cpu` · `gic` · `arm_generic_timer` · `mmu` · `dram_controller` · `uart` · `gpio` · `pmic` · `storage_emmc_ufs` · `gpu_framebuffer` · `device_tree`

Fonte primária: **DTB** em `vendor_boot` / board DT (DTBO sozinho costuma ser overlay incompleto).

## O que isto evita

- Recomeçar drivers do zero sem mapa de endereços  
- Inventar comportamento em estratos sem evidência (fósseis)  
- Misturar “wrap HAL” com “reescrita ISA completa”  
- Assumir que Capstone/MMIO sozinho cobre GIC/timer/PMIC/…  

## O que **não** faz

- Compilar TaurOS / ReactOS  
- Capstone PowerPC completo  
- `generates_os: true` / `auto_fix_complete: true`
- Garantir bases MMIO correctas quando o DT usa `ranges` complexos (validar no DTS)
