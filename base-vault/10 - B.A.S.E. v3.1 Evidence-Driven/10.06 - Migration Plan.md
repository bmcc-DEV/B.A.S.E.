# Migration Plan — v2 → v3.1

> *20 dias para transformar o B.A.S.E. de ML/heurística para Evidence + Contracts + Solver.*

## Passos

| # | Tarefa | Esforço | Crate | Descrição |
|---|--------|---------|-------|-----------|
| 1 | `EvidenceDb` struct + YAML | 2 dias ✅ | `base-core/evidence/` | Struct com fatos puros, serialização YAML |
| 2 | `EvidenceAssessment` | 1 dia ✅ | `base-core/evidence/assessment.rs` | Separado da evidência, confidence + sources |
| 3 | `ReferenceDesign` output | 2 dias ✅ | `base-core/design/` | YAML de saída principal, PCB marcado engineering_draft |
| 3 | Refatorar inference → EvidenceDb | 3 dias | `base-core/inference/` | `extract_blocks` emite `EvidenceDb` em vez de `HardwareSpec` |
| 4 | Purificar BIR para fatos | 2 dias | `base-bir/` | Remover confidence dos tipos BIR |
| 5 | Contract Solver formal | 4 dias ✅ | `base-core/solver/` | `requires` × `provides` matching |
| 6 | ReferenceDesign output | 2 dias | `base-core/design/` | YAML de saída principal |
| 7 | PCB = opcional + draft | 1 dia | `base-pcb/` | Marcar output como "Engineering Draft" |
| 8 | Remover `base-learn/` | 1 dia | — | Remover crate e dependências ML |
| 9 | Atualizar CLI | 2 dias | `base-cli/` | `evidence`, `assess`, `design` comandos |
| 10 | Atualizar vault + README | 2 dias | docs | Documentar nova arquitetura |

## O que é REMOVIDO

| Item | Motivo |
|------|--------|
| `base-learn/` crate | ML substituído por solver formal |
| `base-core/kg/` | Knowledge Graph substituído por Evidence DB |
| `base-core/twin/` | Digital Twin substituído por Contract Verification |
| `confidence` em evidências | Assessment separado |
| Heurística de classificação | Substituído por matching formal |
