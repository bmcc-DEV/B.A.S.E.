# Use Case: Power Mac G5 → NeoG5

## Sistema Original

| Componente | Especificação |
|------------|--------------|
| CPU | 2× PowerPC 970MP @ 2.5GHz |
| RAM | DDR1-400 @ 4GB (8× 512MB) |
| GPU | NVIDIA NV40 (AGP 8x) |
| Áudio | TI TAS3204 (I2S) |
| Storage | SATA 1.5Gbps |
| Ethernet | Broadcom BCM5780 (Gigabit) |
| USB | USB 2.0 |
| PCIe | PCIe 1.0 (via bridge AGP→PCIe) |
| Firmware | OpenFirmware |
| OS | Mac OS X Tiger/Leopard |

## Análise

Complexidade: **Alta** (2 CPUs, GPU dedicada, áudio DSP, OpenFirmware)

## Alvo B.A.S.E.

| Função | Componente B.A.S.E. |
|--------|--------------------|
| CPU | Cortex-A55 @ 2.0GHz (via MMU translation) |
| GPU | RP2350 PIO + SPI display |
| DMA | RP2350 DMA (8 ch) |
| Áudio | I2S via PIO + DAC externo (PCM5102A) |
| RAM | DDR3-1600 (8GB) |
| Storage | SD Card (SPI) |
| Ethernet | WIZnet W5500 (SPI) |
| USB | RP2350 USB 1.1 |

## Desafios Específicos

1. **Dual-core PowerPC → single-core Cortex**: Firmware pode depender de SMP
2. **AGP → SPI**: GPU muito mais lenta, timeouts de polling
3. **OpenFirmware → HAL**: BIOS shim complexo (tabelas ACPI, SMBIOS)
4. **DDR1 → DDR3**: Latências diferentes, memory controller translation
5. **PCIe 1.0 → barramento SPI**: Dispositivos PCIe originais não funcionarão sem stub driver

## Por que este caso?

É o alvo mais complexo e ambicioso. Valida que o B.A.S.E. funciona para sistemas high-end.
