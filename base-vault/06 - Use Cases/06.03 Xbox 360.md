# Use Case: Xbox 360

## Sistema Original

| Componente | Especificação |
|------------|--------------|
| CPU | 3× PowerPC Xenon @ 3.2GHz |
| GPU | Xenos (ATI) 500MHz |
| RAM | 512MB GDDR3 unificada |
| Storage | DVD-ROM + HDD (20-250GB) |
| Rede | 100Mbps Ethernet |
| USB | USB 2.0 |
| Áudio | HDMI/optical + codec interno |
| Chipset | Southbridge Xenos + little-endian mode |

## Análise

Complexidade: **Muito Alta** (3 CPUs, GPU custom, GDDR3 unificada, nGPU+CPU em um package)

## Desafios

1. **CPU tri-core PowerPC → algo moderno**: SMP complexo, barreira de memória
2. **GPU Xenos**: Shader unified, EDRAM de 10MB, lógica de tile-based rendering
3. **Memória unificada**: CPU + GPU compartilham GDDR3 via barramento 128-bit
4. **Proteção**: Xbox 360 tem crypto checks em boot ROM
5. **Timing extremamente apertado**: Jogos otimizados para latências específicas

## Alvo B.A.S.E. (Teórico)

| Função | Abordagem |
|--------|-----------|
| CPU | RP2350 + DMA? Improvável. Cortex-A cluster seria mínimo |
| GPU | FPGA (ECP5) para pipeline Xenos |
| RAM | PSRAM + SDRAM externa |
| Storage | SD Card |
| Rede | W5500 SPI Ethernet |

## Status

**Não recomendado como alvo inicial.** Complexidade muito alta.
Usar como benchmark futuro após validar com [[06.02 Amiga CD32]] e [[06.01 Power Mac G5]].
